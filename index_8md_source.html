6 --------------------------------------------
7 *Meta* is a C++11 tiny metaprogramming library developed by
8 [Eric Niebler](https://github.com/ericniebler).
9 
10 It is released under the Boost Software License and it is header only; that is,
11 to compile with meta you just have to:
12 
13 ~~~~~~~{.cpp}
14 #include <meta/meta.hpp>
15 ~~~~~~~
16 
17 --------------------------------------------
18 
19 The documentation of *Meta* is currently scarce. The best resources are the
20 <a href="group__meta.html">Reference</a> and the
21 <a href="examples.html">Examples</a>.
22 
23 As a motivation and introduction to the library you can read
24 [Eric's original blog post](http://ericniebler.com/2014/11/13/tiny-metaprogramming-library/),
25 but please keep in mind that the library has evolved quite a bit since then.

29 ## Tutorial
30 
31 The tutorial begins with a brief introduction to metafunctions, metafunction
32 classes, and type lists. Then it moves to metafunction composition and
33 currying. Finally, it covers type list algorithms and algorithms for working on
34 integer sequences.

36 ### Metafunction
37 
38 Metafunctions are functions that operate on types. For example,
39 
40 \snippet example/tutorial_snippets.cpp meta_function0
41 
42 is a metafunction taking an arbitrary number of types that always returns
43 void. The return value of a meta function is obtained from a nested type alias
44 called (by convention) `type`. All of the metafunctions in the C++11 standard
45 library follow this convention. C++11 template aliases allow us to write:
46 
47 \snippet example/tutorial_snippets.cpp meta_function1
48 
49 and thus to omit the `typename X::type` when calling a metafunction. C++14
50 standard library provides `_t` metafunction aliases for all the metafunctions in
51 the standard library.
52 
53 *Meta* provides `meta::eval<T>`, which evaluates the metafunction `T` by
54  returning the tested `T::type` alias. This allows metafunction aliases to be
55  written as follows:
56 
57 \snippet example/tutorial_snippets.cpp meta_function2

59 ### Metafunction Class
60 
61 A Metafunction Class is a form of metafunction suitable for higher-order
62 metaprogramming. It is a class with a nested metafunction called (by convention)
63 `apply`:
64 
65 \snippet example/tutorial_snippets.cpp meta_function_class0
66 
67 *Meta* provides the `meta::apply<F, Args...>` metafunction that evaluates the
68  metafunction class `F` with the arguments `Args`:
69 
70 \snippet example/tutorial_snippets.cpp meta_function_class1
71 
72 To turn a metafunction into a metafunction class *Meta* provides the
73 `meta::quote<F>` metafunction:
74 
75 \snippet example/tutorial_snippets.cpp meta_function_class2
76 
77 Note that in the first case we create a metafunction class that will evaluate
78 the metafunction through apply, while in the second case we create a

89 ### Partial function application (currying)
90 
91 Partial application is provided by binding a type to an argument of the

93 type to the first/last argument. That way, we can, e.g., create a metafunction

118 ### Type lists
119 
120 A list of types `Ts...` can be stored in the metafunction
121 `meta::list<Ts...>`. It provides a O(1) member function
122 `meta::list::size()` that returns the size of the list.
123 
124 \snippet example/tutorial_snippets.cpp type_list0
125 
126 As you can see, the `meta::front<List>`, `meta::back<List>`, and
127 `meta::list_element_c<std::size_t, List>` metafunctions provide access to the
128 elements of the list. The `meta::empty<List>` metafunction returns \c true if
129 the list is empty. The `meta::list_element<meta::size_t<N>, List>` metafunction
130 differs from `meta::list_element_c` in that it takes a `meta::size_t<N>`
131 (`std::integral_constant<std::size_t, N>`) insted of an integer:
132 
133 \snippet example/tutorial_snippets.cpp type_list1
134 
135 You can add and remove elements from a list by using the transformation algorithms:
136 
137 \snippet example/tutorial_snippets.cpp type_list2
138 
139 You can concatenate, flatten, and zip multiple lists using
140 `meta::concat<Lists...>`, `meta::join<ListOfLists>`, and `meta::zip<ListOfLists>`:
141 
142 \snippet example/tutorial_snippets.cpp type_list3
143 
144 > TODO: `meta::zip_with` examples
145 
146 Other typical operations on type lists include iteration, reductions, finding
147 elements, removing duplicates:
148 
149 \snippet example/tutorial_snippets.cpp type_list4
150 
151 To convert other type sequences into a `meta::list`, the utility metafunction
152 `meta::as_list<Sequence>` is provided. For example:

154 \snippet example/tutorial_snippets.cpp type_list5

162 ### Overview
163 
164 This is a brief overview of the functionality in meta:
165 
166 - Metafunction: `meta::eval`, `meta::apply`, `meta::defer`, `meta::quote`,
167  `meta::quote_trait`, `meta::always`, `meta::id`,
168  `meta::compose`, `meta::bind_front`, `meta::bind_back`, `meta::curry`,
169  `meta::uncurry`, `meta::lambda`, `meta::let`, `meta::apply_list`.
170 - List: `meta::list`, `meta::front`, `meta::back`, `meta::at`,
171  `meta::at_c`. `meta::empty`, `meta::size`.
172 - Logical: `meta::if_`, `meta::and_`, `meta::or_`, `meta::not_`.
173 - Query and search: `meta::all_of`, `meta::any_of`, `meta::none_of`, `meta::in`,
174  `meta::find`, `meta::reverse_find`, `meta::find_if`, `meta::reverse_find_if`,
175  `meta::cout`.
176 - Transformation: `meta::concat`, `meta::join`, `meta::zip`, `meta::zip_with`,
177  `meta::as_list`, `meta::push_front`, `meta::push_back`, `meta::drop`,
178  `meta::drop_c`, `meta::pop_front`, `meta::fold`, `meta::reverse_fold`,
179  `meta::accumulate`, `meta::unique`, `meta::replace`, `meta::replace_if`,
180  `meta::filter`, `meta::transform`, `meta::reverse`, `meta::cartesian_product`.
181 - Math: `meta::plus` , `meta::minus`, `meta::multiplies`, `meta::divides`,
182  `meta::negate`, `meta::modulus`, `meta::equal_to`, `meta::not_equal_to`,
183  `meta::greater`, `meta::less`, `meta::greater_equal`, `meta::less_equal`,
184  `meta::bit_and`, `meta::bit_or`, `meta::bit_xor`, `meta::bit_not`,
185  `meta::min`, `meta::max`, `meta::inc`, `meta::dec`.
186 - Run time: `meta::for_each`.
187
